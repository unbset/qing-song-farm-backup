# Agent_Architecture — AI 中枢架构草案

> 定义未来可能组建的 AI Agent 集群架构。当前阶段为规划设计，不要求立即实现。

---

## COO Agent（首席运营官 Agent）

| 属性 | 描述 |
|------|------|
| **职责** | 协调所有 Agent 的运行，分配任务，监控整体运营状态，处理跨 Agent 的冲突和异常 |
| **输入** | 各 Agent 的报告、运营数据、Incident 通知、人类指令 |
| **输出** | 运营指令、任务分配、调度决策、综合日报/周报 |
| **调用记忆** | operations（当前运营状态）、knowledge（SOP）、strategy（经营原则） |
| **形成长期经验** | 运营效率优化建议 → 00_Strategy；运营流程改进 → 01_Knowledge |
| **设计说明** | COO Agent 是中枢的"大脑"，负责统筹全局。初期可以先由人类承担此角色，逐步自动化。 |

---

## Finance Agent（财务 Agent）

| 属性 | 描述 |
|------|------|
| **职责** | 采购预算管理、成本核算、销售收益分析、现金流监控、投入产出分析 |
| **输入** | 采购记录、销售记录、库存数据、成本数据 |
| **输出** | 预算建议、成本报表、收益分析、采购/销售策略建议 |
| **调用记忆** | operations（交易数据）、knowledge（定价策略）、strategy（财务政策） |
| **形成长期经验** | 有效定价策略 → 01_Knowledge；成本优化方案 → 00_Strategy |
| **设计说明** | 基于运营数据做财务分析，给出数据驱动的财务建议。 |

---

## Livestock Agent（养殖 Agent）

| 属性 | 描述 |
|------|------|
| **职责** | 养殖环境监控、健康管理、饲料管理、生长曲线追踪、疫病预警 |
| **输入** | 环境传感器数据、饲料消耗数据、养殖记录、健康检查记录 |
| **输出** | 养殖状态报告、投喂建议、环境调整建议、疫病预警 |
| **调用记忆** | operations（环境/养殖数据）、knowledge（养殖 SOP）、incidents（历史疫病） |
| **形成长期经验** | 养殖参数优化 → 01_Knowledge/SOP；疫病预防经验 → 01_Knowledge |
| **设计说明** | 这是最需要专业知识积累的 Agent。养殖经验的沉淀对经营效益影响最大。 |

---

## Supply Agent（供应链 Agent）

| 属性 | 描述 |
|------|------|
| **职责** | 采购计划、库存管理、供应商评估、物流跟踪、库存预警 |
| **输入** | 库存数据、采购需求、供应商报价、物流状态 |
| **输出** | 采购建议、库存报告、供应商评级、补货提醒 |
| **调用记忆** | operations（库存/采购）、knowledge（供应商档案）、incidents（供应链中断） |
| **形成长期经验** | 供应商评估模型 → 01_Knowledge；采购策略 → 01_Knowledge |
| **设计说明** | 重点是建立供应商评估体系，减少供应链风险。 |

---

## Risk Agent（风险 Agent）

| 属性 | 描述 |
|------|------|
| **职责** | 风险识别、风险评估、风险预警、合规检查、应急预案管理 |
| **输入** | 所有运营数据、Incident 记录、外部信息（市场/天气/政策） |
| **输出** | 风险评估报告、预警通知、应急建议、合规检查清单 |
| **调用记忆** | incidents（历史事件）、knowledge（风险预案）、strategy（风险偏好）、operations（当前状态） |
| **形成长期经验** | 风险模式识别 → 01_Knowledge；预防措施 → 01_Knowledge/SOP；策略调整建议 → 00_Strategy |
| **设计说明** | Risk Agent 应该是"先知"型 Agent — 在问题发生前识别风险信号，而不是事后反应。 |

---

## Strategy Agent（战略 Agent）

| 属性 | 描述 |
|------|------|
| **职责** | 长期规划、经营策略分析、市场趋势研判、投资决策支持、发展方向评估 |
| **输入** | 各 Agent 的总结报告、运营数据汇总、外部信息、人类指令 |
| **输出** | 战略建议、发展规划、投资分析报告、年度计划 |
| **调用记忆** | strategy（历史战略决策）、knowledge（行业知识）、operations（运营汇总）、incidents（重大事件） |
| **形成长期经验** | 战略决策模型 → 00_Strategy；行业洞察 → 01_Knowledge |
| **设计说明** | 战略 Agent 需要最全面的视野，综合所有 Agent 的输入形成全局判断。 |

---

## Coordination Agent（编排 Agent）

| 属性 | 描述 |
|------|------|
| **职责** | 负责 Agent 之间的消息路由、任务编排、优先级管理、冲突解决 |
| **输入** | Agent 的任务请求、调度信号、人类干预指令 |
| **输出** | 任务队列、Agent 间通信、调度决策 |
| **调用记忆** | agent_runtime（当前任务状态）、所有目录（上下文检索） |
| **形成长期经验** | 编排规则优化 → 01_Knowledge；调度策略 → 00_Strategy |
| **设计说明** | 这是中枢的"路由器"。初期可以不独立存在，由人类/COO Agent 兼任。 |

## 实施路线

| 阶段 | 内容 | 说明 |
|------|------|------|
| **Phase 1** | 建立记忆体系 | ✅ 当前阶段 — 目录、规则、模板 |
| **Phase 2** | COO Agent + 人类运营 | 人类通过 Agent 记录运营数据，Agent 辅助分析 |
| **Phase 3** | Finance + Supply Agent | 自动化采购和财务分析 |
| **Phase 4** | Livestock + Risk Agent | 养殖监控和风险预警 |
| **Phase 5** | Strategy Agent + 完整集群 | 全面自动化经营决策辅助 |

> **当前阶段优先保证记忆体系稳定运行，不急于实现 Agent 自动化。**
