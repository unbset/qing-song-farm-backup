    言渊，这个问题我想从两个角度回答你。

    现有的分析能力

    你其实已经有一些分析工具了：

    | 已安装的                           | 能力                         |
    |------------------------------------|------------------------------|
    | sn-da-excel-workflow 系列          | Excel 数据分析、统计、可视化 |
    | gbrain-query                       | 知识检索与综合回答           |
    | gbrain-perplexity-research         | 联网调研与增量分析           |
    | engineering/category-statistics 等 | 分类统计、趋势分析           |

    但问题在哪

    这些是"通用"工具。你的农场需要的是懂养殖的分析，比如：


    饲料消耗量 vs 增重比  → 算出料肉比
    月均死亡率 + 根因     → 发现疫病规律
    不同批次出栏效益对比   → 优化采购决策


    通用的 Excel 统计工具做不了这个——它们不懂"料肉比"、"死亡率"、"批次成本"这些农场概念。

    我的建议

    不装现成的，而是写一个定制的 farm-analytics skill。

    内容就是你农场的分析框架：


    Farm Analytics Framework

    指标定义：
      料肉比 = 饲料总消耗(kg) / 总增重(kg)
      月死亡率 = 当月死亡数 / 月初存栏数
      批次利润率 = (销售收入 - 总成本) / 总成本

    分析场景：
      1. 养殖效率分析 → 调 gbrain 计算指标 + hindsight 查历史对比
      2. 成本分析 → 调 gbrain 做成本分解 + hindsight 查历史价格
      3. 趋势预测 → gbrain 做趋势分析
      4. 异常检测 → 数据超出阈值 → 触发 Incident

    输出：
      分析报告 → 写入 02_Operations 或归档到 01_Knowledge


    这样每次你说"分析一下这批的出栏效益"，我就知道：
    1. 查 hindsight 拿这批的成本数据和历史批次对比
    2. 计算料肉比、利润率等农场专属指标
    3. 输出分析报告
    4. 把可复用的结论沉淀到 hindsight
