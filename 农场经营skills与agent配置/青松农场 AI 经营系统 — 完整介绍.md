青松农场 AI 经营系统 — 完整介绍



    一、系统定位

    不是"自动经营农场"，而是"降低经营复杂度，辅助长期经营决策"。

    AI 的职责：记录 → 整理 → 复盘 → 分析 → 提醒 → 协助规划。最终经营决策由人类完成。

    三层架构规则

    | 层     | 组件         | 能做什么                                  | 不能做什么                   |
    |--------|--------------|-------------------------------------------|------------------------------|
    | 调度层 | Hermes（我） | 统一入口、任务分发、记忆写入审批          | 不能替代知识库、不能替代推理 |
    | 推理层 | gbrain       | 策略推理、方案对比、联网调研、风险评估    | ❌ 不能写入任何记忆          |
    | 存储层 | hindsight    | 长期知识存储、SOP、Incident归档、经验沉淀 | 只能存不能分析               |

    核心规则：gbrain 产出是临时思考结果，不自动存；需要沉淀时必须经 Hermes 审批后写入 hindsight。



    三、Hermes 的五顶帽子

    不创建独立 Agent 实体，Hermes 面对不同任务切换对应"帽子"：

    帽子: 🧠 CEO
    职责: 战略分析、经营复盘、风险提醒
    负责: gbrain 深度推理 + 战略决策辅助
    不负责: 日常执行
    典型任务: "分析下季度方向""复盘本月数据"
    ────────────────────────────────────────
    帽子: 🔧 COO
    职责: 运营管理、活动流程、SOP优化
    负责: 项目管理 + 运营数据整理
    不负责: 财务战略、品牌
    典型任务: "优化活动动线""更新SOP"
    ────────────────────────────────────────
    帽子: 👥 CRM
    职责: 用户分析、会员运营、社群
    负责: marketing/sales 类角色
    不负责: 内容创作、定价
    典型任务: "分析复购率""整理用户反馈"
    ────────────────────────────────────────
    帽子: 📱 Content
    职责: 内容策划、短视频、小红书
    负责: marketing 角色 + 品牌定位
    不负责: 经营战略、定价
    典型任务: "想个选题""写拍摄脚本"
    ────────────────────────────────────────
    帽子: 💰 Finance
    职责: 成本分析、收益核算、人效坪效
    负责: finance 类角色 + Excel分析
    不负责: 用户体验、品牌
    典型任务: "算利润率""分析成本结构"

    切换规则：戴上帽子 → 调用对应 skill → 出结果 → 脱帽（不留状态）

    | Skill               | 功能                                                      |
    |---------------------|-----------------------------------------------------------|
    | hindsight-local     | 本地记忆操作：retain（存）recall（查）reflect（综合反思） |

    五、信息流转流程

    标准流程


    输入 → Hermes 判断
      ├── 查历史经验/SOP     → hindsight-local recall → 返回 Hermes → 输出
      ├── 需要推理/分析      → gbrain 对应 skill → 临时结论 → Hermes 综合 → 输出
      ├── 需要专业内容输出   → agency-agents 对应角色 → 产出 → Hermes 整合 → 输出
      ├── 需要多Agent协作    → NEXUS 方法论 → 分阶段推进 → Hermes 协调 → 输出
      └── 可沉淀经验         → Hermes 审批 → hindsight retain → 归档


    经营复盘闭环（核心）


    现实世界经营
        ↓ 填写
    10-经营日志（每天10分钟）
        ↓ 沉淀
    hindsight（结构化存储）
        ↓ 定期分析
    gbrain（周/月复盘）
        ↓ 输出
    SOP优化 / 战略调整（人类决策）
        ↓
    回到现实经营

    七、系统原则

    1. 人类拥有最终决策权 — AI 是辅助工具，不是经营主体
    2. 现实经营优先于 AI 架构 — 先有数据，后有优化
    3. 真实数据优先于理论推演 — 经营日志驱动，不是空想
    4. 长期主义优先于短期优化 — 经验积累比即时 KPI 更重要
    5. 保持轻量、灵活、可调整 — 不自我复杂化，不自动扩张
