# 03_Incidents — 事件分析与复盘记忆

> 事件记忆。支持完整的事件 → 分析 → 解决方案 → 经验总结链路。

## 用途

记录经营中发生的所有异常事件、故障、风险，并从每次事件中提炼经验。

## 内容范畴

- 异常事件报告
- 疫病/病害事件
- 设备故障
- 供应链中断
- 风险分析与评估
- 问题调查过程
- 解决方案与效果追踪
- 结果复盘与经验总结

## 核心流程

```
事件发生 → 记录 Incident → 分析根因 → 制定方案 → 执行 → 复盘 → 经验沉淀到 01_Knowledge
```

**复盘完成后，经验总结必须沉淀到 01_Knowledge 目录。**

## 生命周期

| 状态 | 保留期 | 说明 |
|------|--------|------|
| 处理中 | 持续 | 未关闭的事件保留 |
| 已关闭 | 1 年 | 经验已沉淀到 01_Knowledge 后 |
| 历史归档 | 3 年 | 重要事件的长期参考 |

## 文件命名规范

```
YYYY-MM-DD_事件类型_简述.md
```

示例：
```
2026-05-26_设备故障_3号风机停机.md
2026-06-01_疫病_疑似口蹄疫排查.md
```
