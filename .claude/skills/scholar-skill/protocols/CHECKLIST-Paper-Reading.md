# 论文阅读启动检查清单

**版本**: v2.1  
**最后更新**: 2026-03-18  
**用途**: 每次执行论文阅读任务前必须完成

---

## 📋 启动前检查（必读）

### 1. 确认阅读级别

- [ ] L1 快速筛选（5 分钟）→ P2 论文
- [ ] L2 标准阅读（45 分钟）→ P1 论文
- [ ] L3 精读（2.5 小时）→ P0 论文

**判定依据**:
- P0: 与核心方向直接相关（CAD Generation / Physical AI）
- P1: 方向相关，技术有借鉴价值
- P2: 边缘相关，仅知识拓展

---

### 2. 确认论文可获取

- [ ] PDF 文件已下载 / ArXiv URL 有效
- [ ] 补充材料已获取（如需要）
- [ ] 代码仓库已找到（如需要）

**⚠️ 重要**: 如无法获取完整论文，**跳过而非创建推断性笔记**（学术诚信原则）

---

### 3. 确认 Obsidian 路径

- [ ] Obsidian 仓库路径正确
- [ ] 目标文件夹存在（1-Papers/By-Topic/{Topic}/）
- [ ] 记忆文件夹存在（memory/semantic/, memory/procedural.md, etc.）

---

### 4. 确认时间预算

- [ ] L1: 预留 5 分钟
- [ ] L2: 预留 45 分钟
- [ ] L3: 预留 2.5 小时

**⚠️ 重要**: 不要压缩阅读时间，质量 > 速度

---

### 5. 确认输出要求

#### L1 输出
- [ ] 一句话概括（50 字内）
- [ ] 优先级评级（P0/P1/P2）
- [ ] potential_memory 标记（true/false + reason）

#### L2 输出
- [ ] 标准笔记（3-5KB）
- [ ] Semantic Memories 3-5 条
- [ ] Episodic Memories 1-2 条
- [ ] Procedural Memories 1 条候选
- [ ] 每条记忆有 action 字段
- [ ] 给出与旧知识的连接关系（允许 standalone）

#### L3 输出
- [ ] L2 全部要求满足
- [ ] 深度笔记（10-15KB）
- [ ] 完成常驻/检索记忆划分
- [ ] 至少修订或评估 1 组旧知识
- [ ] 至少提炼 2-3 条程序规则
- [ ] 标记冲突、时间演化或废弃知识
- [ ] 回答"这篇论文如何改变我的知识结构"

---

### 6. 确认工具就绪

- [ ] arxiv-watcher / academic-research-hub 可用
- [ ] obsidian-direct 可用
- [ ] tavily_extract 可用（如需要网页提取）
- [ ] pdf 工具可用（如需要 PDF 解析）

---

### 7. 确认通知设置

- [ ] 飞书推送已配置（如需要）
- [ ] L3 里程碑推送已启用（25%/50%/75% 进度）
- [ ] 完成报告推送已启用

---

## ✅ 启动确认

所有检查项完成后，回复：

```
✅ 检查清单完成
- 阅读级别：L{1|2|3}
- 优先级：P{0|1|2}
- 时间预算：{5|45|150} 分钟
- 论文可获取：是/否
- 开始执行阅读
```

---

## 🚨 异常处理

### 论文无法获取
- 处理：跳过并记录原因，不创建推断性笔记
- 汇报：告知主人论文不可获取的原因

### Obsidian 路径不存在
- 处理：自动创建目录结构
- 汇报：完成后告知主人

### 记忆抽取失败
- 处理：降级到 L1 输出，记录错误日志
- 汇报：分析失败原因，建议修复方案

### 知识冲突检测
- 处理：标记为 `contradict`，等待周巩固时裁决
- 汇报：如高冲突（核心范式），立即汇报主人
