# Memory_Rules — 记忆规则

> 定义什么信息写入长期记忆、什么信息禁止写入、以及各类记忆的生命周期管理规则。

---

## 1. 允许写入长期记忆的信息

以下信息有资格进入长期记忆（00_Strategy 或 01_Knowledge）：

### 1.1 经营战略类
- 农场定位、愿景、长期目标
- 重大战略决策及其依据
- 风险偏好与风控策略
- 发展规划与路线图
- 经营原则与价值观

### 1.2 知识与流程类
- SOP（标准操作流程）
- 经过验证的最佳实践
- 设备操作规范与维护手册
- 供应商评估与档案
- 通过复盘沉淀的经验总结
- 行业研究、政策法规信息

### 1.3 事件复盘类
经过完整复盘确认的经验总结（从 03_Incidents 沉淀而来），包括：
- 根因分析结论
- 验证有效的解决方案
- 预防措施建议
- 改进后的流程/SOP

---

## 2. 禁止写入长期记忆的信息

以下信息**不得**写入长期记忆（00_Strategy 或 01_Knowledge）：

### 2.1 绝对禁止
- 个人身份敏感信息（身份证号、银行账号、密码等）
- 未经验证的猜测（除非标注"待验证"且放入临时区域）
- 重复已有长期记忆的冗余内容
- 临时计算中间结果
- Agent 内部调试日志
- 聊天记录原文（非经营内容）

### 2.2 应当清理的信息
- 过期超过 90 天的运营日志
- 已归档超过 1 年的普通事件记录
- Agent 运行日志超过 7 天
- 已合并/去重的冗余文件

---

## 3. 长期 / 短期记忆分类规则

| 类别 | 目录 | 保留期 | 清理机制 |
|------|------|--------|----------|
| 永久长期 | 00_Strategy | 永久 | 手动归档 |
| 长期知识 | 01_Knowledge | 长期 | 版本替代后手动归档 |
| 短中期运营 | 02_Operations | 30-90 天 | 自动清理 / 月度汇总后清 |
| 事件分析 | 03_Incidents | 关闭后 1 年 | 复盘完成后自动标记 |
| 短期运行 | 04_Agent_Runtime | 7-30 天 | 自动清理 |

**判断流程：**

```
有新信息 →
  是战略/原则/定位？→ 00_Strategy（永久）
  是知识/SOP/经验？→ 01_Knowledge（长期）
  是日常运营数据？→ 02_Operations（短期）
  是异常/故障/风险？→ 03_Incidents（中长期）
  是 Agent 运行记录？→ 04_Agent_Runtime（短期）
```

---

## 4. 自动总结规则

### 4.1 触发条件
- 02_Operations 积累超过 30 条记录 → 触发月度运营总结
- 03_Incidents 积累超过 5 条已关闭事件 → 触发季度复盘汇总
- 04_Agent_Runtime 积累超过 50 条日志 → 触发性能分析

### 4.2 总结输出
- 月度运营总结 → 提炼规律 → 写入 01_Knowledge
- 季度复盘汇总 → 提取共性根因 → 更新相关 SOP
- Agent 性能分析 → 优化建议 → 写入 00_Strategy 或 04_Agent_Runtime

### 4.3 总结原则
- 总结必须可追溯（引用源文件）
- 总结必须标注生成时间
- 总结不能替代源数据（源数据按生命周期清理）

---

## 5. SOP 沉淀规则

### 5.1 沉淀触发条件
- Incident 复盘确认有效方案
- 运营中发现更优操作流程
- 设备/工艺变更
- 季度总结确认的最佳实践

### 5.2 版本管理
```
SOP_主题_v1.md
SOP_主题_v2.md  ← 更新说明：根据 Incident XXX 复盘改进
SOP_主题_v3.md  ← 更新说明：根据季度总结优化
```

### 5.3 质量要求
- 步骤清晰、可执行
- 包含预期结果和验收标准
- 标注适用范围和前提条件
- 注明创建人/创建时间/版本号

---

## 6. Incident 复盘规则

### 6.1 必须复盘的事件类型
- 疫病/健康事件
- 设备故障导致生产中断
- 供应链断裂
- 重大经济损失
- 重复出现 2 次以上的同类问题
- 安全事件

### 6.2 复盘必须回答的问题
1. 发生了什么？
2. 为什么会发生（根因）？
3. 当时如何处理的？
4. 什么方法有效？什么无效？
5. 如何防止再次发生？
6. 哪些 SOP 需要更新？

### 6.3 复盘完成标准
- [ ] 根因已确认
- [ ] 解决方案已验证有效
- [ ] 经验总结已写入 01_Knowledge
- [ ] 相关 SOP 已更新（如果需要）

---

## 7. Agent 运行日志保留规则

### 7.1 保留期限
| 日志类型 | 保留期 | 说明 |
|----------|--------|------|
| 普通执行日志 | 7 天 | 自动清理 |
| 失败/异常日志 | 30 天 | 分析后清理 |
| 关键决策推理 | 归档到 00_Strategy | 转入长期 |
| Memory 调用记录 | 7 天 | 仅用于调试 |

### 7.2 必须保留的日志
- Agent 做出的经营决策及其推理过程
- Agent 发现的异常和触发 Incident 的记录
- Agent 执行失败案例（用于改进）

### 7.3 应当清理的日志
- 正常执行完成的例行任务日志
- 成功调用的 Memory 操作记录
- 重复的轮询/状态检查记录
