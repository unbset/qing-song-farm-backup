# Memory_Namespace_Map — 记忆命名空间体系

> 定义系统中所有记忆命名空间及其用途、生命周期、检索优先级。

---

## 命名空间一览

| Namespace | 目录 | 用途 | 生命周期 | 检索优先级 | 自动总结 | 长期保留 |
|-----------|------|------|----------|------------|----------|----------|
| `strategy` | 00_Strategy | 长期战略与经营原则 | 永久 | ⭐⭐⭐⭐⭐ 最高 | ❌ 手动 | ✅ 是 |
| `knowledge` | 01_Knowledge | SOP、经验、最佳实践 | 长期 | ⭐⭐⭐⭐ 高 | ✅ 季度 | ✅ 是 |
| `operations` | 02_Operations | 日常运营数据 | 30-90 天 | ⭐⭐⭐ 中 | ✅ 月度 | ❌ 否 |
| `incidents` | 03_Incidents | 异常事件与复盘 | 关闭后 1 年 | ⭐⭐⭐⭐ 高 | ✅ 季度 | ✅ 是（经验部分） |
| `agent_runtime` | 04_Agent_Runtime | Agent 运行日志 | 7-30 天 | ⭐⭐ 低 | ✅ 月度 | ❌ 否（决策例外） |

---

## strategy — 战略命名空间

| 属性 | 值 |
|------|-----|
| **用途** | 存储农场经营战略、定位、原则、风险偏好、发展规划 |
| **生命周期** | **永久保留**，不参与任何自动清理 |
| **检索优先级** | 最高。在所有经营决策中优先检索 |
| **自动总结** | 不允许自动总结。战略信息只能由人类手动整理 |
| **长期保留** | 是。这是系统的"宪法"级记忆 |
| **写入条件** | 仅当信息涉及长期经营方向、原则、风险政策时写入 |
| **标签规范** | `strategy`, `principle`, `risk`, `roadmap`, `vision` |

---

## knowledge — 知识命名空间

| 属性 | 值 |
|------|-----|
| **用途** | 存储 SOP、最佳实践、养殖经验、设备知识、供应商档案 |
| **生命周期** | **长期保留**，通过版本迭代更新 |
| **检索优先级** | 高。日常经营中最常查询的命名空间 |
| **自动总结** | 允许季度总结，但需人类审核后写入 |
| **长期保留** | 是。只有被明确替代的知识才归档 |
| **写入条件** | 经过验证的经验、经过复盘确认的结论、经过审批的 SOP |
| **标签规范** | `knowledge`, `sop`, `best_practice`, `equipment`, `supplier`, `经验` |

---

## operations — 运营命名空间

| 属性 | 值 |
|------|-----|
| **用途** | 存储日常采购、库存、销售、环境数据、任务记录、运营日志 |
| **生命周期** | **30-90 天**，定期自动清理 |
| **检索优先级** | 中。当前运营状态查询 |
| **自动总结** | 允许月度自动总结，总结结果提炼后写入 knowledge |
| **长期保留** | 否。运营数据是原料，不是成品 |
| **写入条件** | 日常运营记录均可写入 |
| **清理规则** | 30 天日志 / 90 天交易记录 / 月度汇总后清 |

---

## incidents — 事件命名空间

| 属性 | 值 |
|------|-----|
| **用途** | 存储异常事件、疫病、设备故障、风险分析、复盘记录 |
| **生命周期** | **关闭后 1 年**，经验部分转移至 knowledge 后标记 |
| **检索优先级** | 高。风险管理和问题排查优先检索 |
| **自动总结** | 允许季度复盘总结，共性根因更新到知识库 |
| **长期保留** | 经验总结部分长期保留（转移到 knowledge），原始事件记录按生命周期清理 |
| **写入条件** | 任何经营异常事件必须写入 |
| **复盘要求** | 复盘完成后，经验总结必须写入 knowledge |

---

## agent_runtime — Agent 运行命名空间

| 属性 | 值 |
|------|-----|
| **用途** | 存储 Agent 执行日志、工作流状态、调度记录、失败案例 |
| **生命周期** | **7-30 天**，短周期自动清理 |
| **检索优先级** | 低。仅在 Agent 自检或调试时使用 |
| **自动总结** | 允许月度性能分析，优化建议写入 strategy 或 knowledge |
| **长期保留** | 否。关键决策推理可归档到 strategy，其余清理 |
| **写入条件** | Agent 自动写入，人类无需手动管理 |
| **清理规则** | 普通日志 7 天 / 失败案例 30 天 / 决策推理归档 |
