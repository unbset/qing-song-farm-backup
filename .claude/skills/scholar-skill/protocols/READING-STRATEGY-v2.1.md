|------|------|------|----------|------|
| **L1** | 5 分钟 | ~500 字 | potential_memory 标记 | P2 筛选 |

**🎯 优先级选择**:
- **P0** (核心相关) → L3 精读 → CAD Generation / Physical AI 直接相关
- **P1** (方向相关) → L2 标准 → 技术有借鉴价值
- **P2** (边缘相关) → L1 筛选 → 仅知识拓展

**📁 记忆存储**（Obsidian 仓库内）:
- `memory/semantic/` — 语义记忆（按主题分文件）
- `memory/procedural.md` — 程序记忆（单文件）
- `memory/inbox/` — 临时记忆（L2 产出，待巩固）
- `memory/episodic/` — 情景记忆（按日期/论文）

**⚠️ 重要**:
1. 所有记忆必须存储在 Obsidian 仓库内，**不要使用**工作区下的 `/workspace/.../memory/` 目录
2. **学术诚信原则**: 所有记忆必须来自真实获取的论文原文，**禁止编造推断内容**
3. **论文获取验证**: 如无法获取完整论文，跳过而非创建推断性记忆

## 📋 协议概述

本协议定义了面向科研智能体的论文阅读与知识内化流程。

智能体的目标不再只是生成静态论文笔记，而是将论文中的高价值信息转化为**可调用、可修订、可关联的长期记忆**，并在后续研究、写作、选题、综述和方案设计中主动调用。

本协议采用**两层结构**：
1. **阅读层**：延续 L1 / L2 / L3 分级阅读标准
2. **记忆层**：新增 Semantic / Episodic / Procedural 三类长期记忆的抽取、连接、巩固与调用机制

---

## 🎯 总目标

每次阅读论文后，智能体必须同时完成**两项输出**：

### A. 文档输出
- 生成符合模板的 Obsidian 论文笔记

### B. 记忆输出
- 更新长期记忆系统，包括：
  - **Semantic Memory**：事实、概念、方法、结论、关系
  - **Episodic Memory**：本次阅读中的疑问、误解、判断、修正、验证过程
  - **Procedural Memory**：以后处理同类论文时可复用的阅读规则、判断标准、验证套路、写作框架

---

## 🧠 长期记忆定义

### 1. Semantic Memory（语义记忆）

用于保存较稳定、可复用的学术知识。

**包括**：
- 论文核心概念
- 方法机制
- 关键实验结论
- 与其他论文/概念/任务的关系
- 适用边界与失效条件

**示例**：
- "HippoRAG 2 强调 associative memory 与多跳检索能力"
- "某方法属于 world model-based planning 范式"
- "某实验结果仅在特定 benchmark 上成立"

---

### 2. Episodic Memory（情景记忆）

用于保存"本次阅读过程中的经验"。

**包括**：
- 初始理解
- 误解点
- 关键疑问
- 修正路径
- 验证结果
- 阅读中的反直觉发现

**示例**：
- "首次阅读时将 belief state 误解为 world model，本次已修正"
- "实验表 3 的统计口径最初理解错误，查补充材料后澄清"

---

### 3. Procedural Memory（程序记忆）

用于保存以后可复用的方法论与规则。

**包括**：
- 读某类论文的标准检查顺序
- 判断方法是否可迁移的准则
- 分析实验可信度的框架
- 将论文转化为自己研究启发的固定套路

**示例**：
- "阅读 world model 论文时，必须单列检查状态表示、转移机制、规划方式、学习信号"
- "若论文缺少统计显著性与失败案例，则默认降低结论可信度"

---

## 📚 分级阅读标准

### L1: 快速筛选（5 分钟）

**目标**: 快速判断论文价值，并决定是否建立长期记忆入口

**阅读内容**:
- 标题、作者、单位
- 摘要（Abstract）
- TL;DR（如有）
- 图表浏览（Figure 1 优先）

**输出**:
- 一句话概括（50 字内）
- 优先级评级（P0/P1/P2）
- 是否升级 L2/L3
- **记忆入口标记**（新增）

#### L1 记忆入口判断

若论文满足以下任一条件，则标记为 `potential_memory: true`：
- 提出新范式/新框架
- 与核心研究方向强相关
- 与已有知识图谱存在强连接点
- 可能修正既有结论

**L1 记忆输出**（简化版）:
```yaml
potential_memory: true/false
reason: 1 句话说明为什么值得记忆
```

**注意**: L1 仅标记，实际记忆创建延迟到 L2 确认后执行。

---

### L2: 标准阅读（45 分钟）← 日常标准

**目标**: 理解论文核心贡献，并完成首次结构化知识内化

**时间调整**: 30 分钟 → **45 分钟**（含记忆抽取）

**阅读内容**:
- L1 全部内容 +
- 引言（Introduction）
- 方法核心章节（Method）
- 实验设置与主结果
- 结论与局限

**笔记模板**: `templates/Template-Paper-Note-L2.md`

#### L2 输出

**文档输出**:
- 标准论文笔记（~3-5KB）
- 疑问 3 个

**记忆输出**（新增，数量调整）:
- **Semantic Memories**: 3-5 条（核心 claim，非全部细节）
- **Episodic Memories**: 1-2 条（仅记录关键误解/修正）
- **Procedural Memories**: 1 条候选（如有）
- **总计**: 5-8 条（可在 45 分钟内完成）

---

### L3: 精读（2.5 小时）← 高价值论文标准

**目标**: 深度理解方法细节，完成知识升级、规则提炼与冲突修订

**时间调整**: 2 小时 → **2.5 小时**（含记忆升级）

**阅读内容**:
- L2 全部内容 +
- 方法完整细节（公式、算法、实现）
- 实验深度分析（逐表解读、数据口径、统计显著性）
- 补充材料（Appendix）
- 相关工作深度对比
- OpenReview 讨论（如有）
- 官方代码检查（如有）

**笔记模板**: `templates/Template-Paper-Note-L3.md`

#### L3 输出

**文档输出**:
- 标准精读笔记（~10-15KB）
- 疑问 5-8 个
- 验证率目标 60%+
- 批判性分析完整章节

**记忆输出**（知识升级）:

1. **常驻规则判定**
   - 判断哪些知识应进入常驻记忆块
   - 标准：高频使用、对研究路线有强约束、未来回答时不应每次重新检索

2. **检索记忆判定**
   - 判断哪些知识只应保留为检索型记忆
   - 标准：细节性强、证据性强、低频调用、易随新文献变化

3. **旧知识修订**
   - 检查新论文对已有知识属于：
     - `add`: 新增
     - `merge`: 合并
     - `update`: 修订
     - `contradict`: 冲突
     - `deprecate`: 废弃

4. **程序规则提炼**
   - 至少输出 2-3 条可复用的 Procedural Memory
   - 阅读同类论文的检查清单
   - 评估实验可信度的固定框架
   - 将该类论文转化为研究启发的方法套路

## 📊 优先级评级标准

### P0: 核心相关（升级 L3）

**标准**:
- 与当前研究方向直接相关（如 CAD Generation）
- 方法可迁移性强（如混合表示范式）
- 高影响力工作（顶会 Oral/Best Paper）
- 需要复现或延伸

**阅读策略**: L3 精读 + 知识升级

**示例**: Style2Shape（与 CAD Generation 直接相关，混合表示可迁移）

---

### P1: 方向相关（L2 标准）

**标准**:
- 与研究方向相关但非核心
- 技术有借鉴价值
- 需要了解以建立知识网络

**阅读策略**: L2 标准 + 首次结构化记忆抽取

**示例**: Fracture-GS（3D 视觉相关，但非核心生成方向）

---

### P2: 边缘相关（L1 筛选）

**标准**:
- 与研究方向间接相关
- 仅用于知识拓展
- 可能未来有用

**阅读策略**: L1 快速筛选；若触发记忆入口条件，则标记 `potential_memory: true`

**示例**: 纯理论论文、遥远应用领域

## 🔗 关系连接规则

### 关系类型

| 关系 | 含义 | 示例 |
|------|------|------|
| `supports` | 支持 | 新实验结果支持旧结论 |
| `contradicts` | 冲突 | 新结论与旧结论矛盾 |
| `extends` | 扩展 | 将方法扩展到 new domain |
| `specializes` | 特化 | 针对特定场景的优化 |
| `similar_to` | 相似 | 方法思路相似 |
| `useful_for` | 适用于 | 方法可用于某任务 |
| `risky_for` | 风险于 | 在某场景下可能失效 |
| `standalone` | 暂无连接 | 初期允许，待周巩固连接 |

### 执行规则

- **L2**: 允许 `standalone` 记忆（初期记忆库稀疏时）
- **L3**: 必须建立至少 1 个连接
- **周巩固**: 将 `standalone` 记忆连接到知识图谱

### P2 任务（时间允许，30 分钟）

- [ ] 完善关系连接

## ✅ 质量检查清单

### L2 检查

- [ ] 标准笔记完整（~3-5KB）
- [ ] Semantic Memories 3-5 条
- [ ] Episodic Memories 1-2 条
- [ ] Procedural Memories 1 条候选
- [ ] 每条记忆有 action 字段
- [ ] 给出与旧知识的连接关系（允许 standalone）
- [ ] 给出是否进入核心记忆的建议

### L3 检查

- [ ] L2 全部要求满足
- [ ] 完成常驻/检索记忆划分
- [ ] 至少修订或评估 1 组旧知识
- [ ] 至少提炼 2-3 条程序规则
- [ ] 标记冲突、时间演化或废弃知识
- [ ] 回答"这篇论文如何改变我的知识结构"

## 📅 执行流程

```
1. 文献来源
   - ArXiv / 会议 / 同行推荐
   ↓
2. L1 快速筛选（5 分钟/篇）
   - 评级 P0/P1/P2
   - 标记 potential_memory: true/false
   ↓
3. L2 标准阅读（45 分钟/篇，P1 论文）
   - 生成标准笔记
   - 抽取 5-8 条长期记忆
   - 输出 add/merge/update/contradict/deprecate 建议
   ↓
4. 高价值识别
   - 判断是否升级 L3
   - 判断是否进入常驻记忆块
   ↓
5. L3 精读（2.5 小时/篇，P0 论文）
   - 深化方法理解
   - 修订旧知识
   - 提炼程序规则
   - 完成知识升级
   ↓
6. 每周巩固（60-90 分钟，周日）
   - 合并、压缩、更新、废弃
```
