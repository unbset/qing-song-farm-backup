## 框架结构说明

本框架采用 **“1+N” 双层分析结构**：

-   **1** = 阿米巴单元总体分析（使用七部分主框架）
-   **N** = 该阿米巴下每个产品的独立分析（使用产品分析子框架）

**使用顺序**：
1.  先完成阿米巴总体分析（第一至第七部分）
2.  再列出该阿米巴的产品清单
3.  对每个产品，逐项填写产品分析子框架
4.  将所有产品的财务数据汇总，回填到阿米巴总体分析的第四部分


## 青松农场业务板块分析框架（V1.0）

> **定义**：一个业务板块 = 一个独立的阿米巴单元。    
	一个阿米巴下面可以包含多个产品线    
	比如“酵素+种养循环”是一个阿米巴，它下面的产品包括：蔬菜种植、主粮种植、水果种植、林下养殖（鸡/猪/鹅）、酵素生产、农产品加工（减脂蔬菜包、鸡肉包等）、菜园租赁、动物认养德国。
> **分析对象**：一个独立的阿米巴单元  
> **分析结构**：阿米巴总体分析（七部分） + 产品清单 + 产品分析子框架  
> **核心理念**：每个阿米巴是飞轮上的一个齿轮，每个产品是齿轮上的一个齿


### 第一部分：业务定位分析（战略层）

**这部分回答：这个阿米巴在飞轮里到底干什么？**

| 分析项         | 内容                                                                        | 必填/可选 |
| :---------- | :------------------------------------------------------------------------ | :---- |
| **阿米巴名称**   |                                                                           | 必填    |
| **在飞轮中的位置** | 该阿米巴存在的意义是什么？属于五类经营作用中的哪一类？（引流入口/信任引擎/复购锚点/利润中心/传播杠杆）                     | 必填    |
| **主定位**     | 五类中只能选一个主定位。一个阿米巴可以同时起多重作用，但主定位必须唯一。这是后续所有决策的基准                           | 必填    |
| **辅助作用**    | 除了主定位，还承担哪些次要作用？                                                          | 必填    |
| **业务等级**    | S级—生死业务（缺失则农场不成立）；A级—核心业务（飞轮关键齿轮）；B级—增强业务（提升体验和效率）；C级—锦上添花（有更好，没有也能活）     | 必填    |
| **承接谁**     | 这个阿米巴的客户、信任、流量，主要从哪个上游阿米巴来？                                               | 必填    |
| **输送谁**     | 这个阿米巴的产出（客户信任、复访习惯、现金流），主要输送给哪个下游阿米巴？                                     | 必填    |
| **如果缺失会怎样** | 飞轮中如果没有这个阿米巴，哪一环会断裂？                                                      | 必填    |
| **替代方案**    | 如果这个阿米巴做不起来，有没有其他阿米巴或外部合作可以替代它的功能？                                        | 必填    |
| **现在该不该做**  | 定位该阿米巴的生命周期：初创期（活下来，跑通最小闭环）；成长期（建飞轮，验证上下游衔接）；稳定期（做利润，优化效率）；扩张期（做复制，标准化输出） | 必填    |

**设计逻辑**：分析任何阿米巴，先不要问赚不赚钱，先问：“它在整个农场系统中承担什么角色？”不是每个阿米巴都赚钱，但每个阿米巴都必须为系统贡献价值。所有阿米巴共同形成一个飞轮，才真正构成农场经营系统。
一个阿米巴如果回答不了“承接谁”和“输送谁”，说明它在飞轮中是孤岛，要么重新设计它与上下游的关系，要么砍掉。


### 第二部分：产品与客户分析（需求层）

**这部分回答：卖给谁？卖什么？为什么买？**

| 分析项            | 内容                                                                                                                                  |
| :------------- | :---------------------------------------------------------------------------------------------------------------------------------- |
| **目标客群画像**     | 谁是这个阿米巴的核心客户？画像越具体越好。年龄、家庭结构、消费能力、痛点需求                                                                                              |
| **客户真正买的是什么**  | 用户买的不是产品，而是情绪、身份、健康、体验、想象、价值认同。请明确描述                                                                                                |
| **客户决策逻辑**     | 客户为什么需要这个产品？他们现在用什么替代方案？选择我们的第一推动力是什么？                                                                                              |
| **产品层级设计**     | 列出该阿米巴下所有产品，按层级分类：<br>**引流款**：低价/免费，降低决策门槛，负责“来”<br>**核心款**：主成交产品，体现核心价值，负责“赚”<br>**利润款**：高毛利产品，主利润来源<br>**锚点款**：锁定长期关系，周期性回访，负责“留” |
| **定价逻辑**       | 基于什么定价？成本加成？市场对标？价值定价？为什么客户愿意付这个价格？是否有升级路径？低频高客单还是高频低客单？                                                                            |
| **与农场核心价值的关系** | 这些产品如何体现“修复人与自然的关系”？如果体现不了，是否偏离了农场第一性原理？                                                                                            |
| **是否可标准化**     | 未来扩张的关键。分析能否复制、是否依赖某个人                                                                                                              |
| **用户为什么复购**    | 这是最重要的问题。若回答不上来，这就是一个短命产品（临时活动除外）                                                                                                   |
| **有没有自传播能力**   | 微度假行业极度依赖传播。必须分析有没有拍照点、有没有故事感、有没有冲突感，用户愿不愿意发朋友圈                                                                                     |

**设计逻辑**：产品层级设计是强制要求。一个阿米巴如果只有一个产品、一个价格，抗风险能力极差。


### 第三部分：运营与标准化分析（执行层）

**这部分回答：日常怎么跑？依赖谁？**

| 分析项 | 内容 |
| :--- | :--- |
| **端到端流程** | 从客户接触到服务完成的完整步骤。用5-8个步骤描述清楚 |
| **运营强度** | 极高/高/中/低。该阿米巴日常运营对体力和精力的消耗程度 |
| **关键触点** | 流程中哪些触点决定了客户体验的成败？（通常不超过3个） |
| **人员需求** | 最少需要几个人？什么技能？全职还是兼职？对主理人的人格要求标准（极高/高/中/低） |
| **标准化程度评估** | 这个阿米巴的交付，多大程度上可以靠SOP保证？多大程度依赖人的经验和热情？（用百分比估算） |
| **关键岗位依赖度** | 如果某个岗位的人离职，业务能否在1周内由他人接手？如果能，需要什么条件？ |
| **培训成本** | 一个新人从入职到独立上岗，需要几天？培训材料是否已准备好？ |
| **SOP状态** | 是否容易标准化？是否有书面SOP？是否经过实战验证和迭代？ |

**设计逻辑**：标准化程度低于60%的阿米巴，标记为“高风险”。关键岗位依赖度高的，必须有交叉培训和应急替补方案。


### 第四部分：财务分析（生存层）

**这部分回答：要花多少钱？钱从哪来？什么时候能赚回来？必须极其理性。**

| 分析项 | 内容 |
| :--- | :--- |
| **初始投入** | 列出所有一次性投入项目及金额 |
| **月度固定成本** | 列出每月必须支付的费用：租金、人工、水电、折旧、物料、维护等 |
| **月度变动成本** | 随业务量变化的成本：食材、包材、活动耗材、运输等 |
| **收入项拆解** | 每一项收入的测算逻辑、单价、预期量、季节性波动。汇总该阿米巴下所有产品的收入 |
| **盈亏平衡点** | 月收入达到多少时，能覆盖月成本？必须算，否则容易自我感动 |
| **投资回收期** | 初始投入 ÷ 月均净利润 |
| **季节性影响** | 一年中有几个月是旺季？几个月是淡季？淡季收入能覆盖成本吗？如果不能，缺口多大？ |
| **内部转移定价** | 如果这个阿米巴向农场其他阿米巴提供产品或服务，内部结算价是多少？ |
| **阿米巴损益表** | 月度汇总：收入（外部销售 + 内部转移收入）- 成本（固定 + 变动）= 结余 |
| **现金流平衡** | 高频收入与低频收入相结合，不可过度依赖某一项。日现金流（餐饮）、周现金流（活动）、月现金流、年现金流（会员）需形成互补 |

**设计逻辑**：
-   必须区分旺季和淡季分别测算。用全年平均数掩盖季节性风险的测算，直接打回重做。
-   内部转移定价是阿米巴模式的核心。没有内部定价，各板块协作就变成“人情账”，迟早出矛盾。
-   对于利润型阿米巴，盈亏平衡点是生死线。12个月内达不到盈亏平衡，启动决策需重新评估。


### 第五部分：风险评估（底线层）

**这部分回答：最坏的情况是什么？能承受吗？**

| 分析项 | 内容 |
| :--- | :--- |
| **风险清单** | 列出所有可能导致该阿米巴失败或严重受损的风险事件 |
| **最坏情况推演** | 假设每个风险事件**一定发生**，推演具体后果——损失多少钱？影响多少客户？修复需要多久？是否会导致品牌崩塌？ |
| **承受能力判断** | 最坏情况的后果，农场能承受吗？“能”——继续，设好预案。“不能”——这个风险必须通过事前措施降到“能承受”的级别，否则该阿米巴不能启动 |
| **预防措施** | 针对“不能承受”的风险，具体做什么来降低发生可能性？包括制度、SOP、法律、合同、应急方案等 |
| **应急预案** | 万一发生了，具体做什么来降低损失？谁负责？第几分钟/第几小时做什么？ |
| **退出标准** | 设定量化指标。连续几个月达到什么条件，就暂停或关闭这个阿米巴？ |

**设计逻辑**：
-   不做主观概率评分。人是乐观的动物，会系统性低估坏事发生的概率。
-   “承受能力判断”是核心。不是问“这件事会不会发生”，而是问“发生了我能不能活下来”。
-   退出标准必须提前设定。人会在亏损的业务上越陷越深，因为“已经投入这么多了”。提前写好退出的量化标准，是对抗沉没成本谬误的唯一办法。


### 第六部分：数据与AI融合（加速层）

**这部分回答：采集什么数据？AI能帮什么忙？**

| 分析项 | 内容 |
| :--- | :--- |
| **核心数据项** | 为支撑该阿米巴的3个核心KPI，需要采集哪些原始数据？ |
| **采集方式** | 怎么采？谁采？什么频率？工具是什么？ |
| **AI辅助场景** | AI系统具体能在这个阿米巴中做什么？（SOP辅助生成、数据分析、客户维护提醒、内容选题建议——选当前阶段能落地的，不贪多） |
| **经验沉淀计划** | 这个阿米巴中产生的经验教训，如何进入hindsight？多久复盘一次？ |

**设计逻辑**：
-   只采集“一定会用到的数据”。为“未来可能有用”而采集的数据，永远不会被使用，只会变成填表负担。
-   AI辅助只保留3类：SOP生成与迭代、数据分析与异常标记、客户管理提醒。其他都是远期规划。


### 第七部分：决策矩阵（行动层）

**这部分回答：做不做？什么时候做？什么时候停？**

| 决策节点     | 触发条件                | 决策动作      |
| :------- | :------------------ | :-------- |
| **启动条件** | 满足什么前置条件，这个阿米巴才能启动？ | 启动        |
| **成功标准** | 运营多长时间后，达到什么指标算成功？  | 继续运营，考虑扩展 |
| **预警线**  | 什么指标连续不达标触发预警？      | 启动复盘和调整   |
| **退出条件** | 什么情况下关闭或转让这个阿米巴？    | 执行退出      |

**设计逻辑**：
-   启动条件、成功标准、预警线、退出条件——这四个节点必须全部量化。
-   “看情况再说”是最危险的决策方式。人在情境中会找各种理由说服自己“再坚持一下”。

## 产品分析子框架（V3.0）

> **定位**：用于分析一个阿米巴单元下的每一个产品  
> **结构**：表A（产品体检表）→ 表B（产品落地规划表）  
> **使用顺序**：先用表A快速筛查，不通过直接砍掉；全部通过后，用表B做详细落地设计  
> **核心原则**：表A管生杀，表B管执行。两项不重复，表A结论直接带入表B

### 表A：产品体检表（快速筛查——能不能做）

> **用途**：新产品想法产生后，用本表做生死判断。  
> **通过标准**：10项中任何一项答案是“没有”或“说不清”，该产品砍掉或回炉。必须全部通过才能进入表B。

| 序号  | 体检项目  | 体检内容                                                                       | 体检结果 |
| --- | ----- | -------------------------------------------------------------------------- | ---- |
| 1   | 用户定义  | 谁是这个产品的目标用户？（定性/定量）<br>谁是这个产品的目标客户？（定性/定量）                                 |      |
| 2   | 需求定义  | 他们的痛点是什么？<br>他们的痒点是什么？<br>他们的嗨点是什么？                                        |      |
| 3   | 解决方案  | 这个产品如何解决他们的需求？（直击痛点/锁定嗨点）                                                  |      |
| 4   | 竞争力定义 | 所提供产品的差异化是什么（微创新/开创新品类）？为什么客户选我们而不是替代方案？                                   |      |
| 5   | 支持系统  | 交付这个产品，我们的核心能力是什么？<br>关键资源是什么？                                             |      |
| 6   | 排兵布阵  | 战场在哪（本地/线上/私域）？用什么武器（短视频/社群/体验课）触达用户？                                      |      |
| 7   | 战果复盘  | 样板测试做了吗？试错闭环跑通了吗？商业模型、能力产品化模型、流量模块化模型、流程标准化模型、商业私域化模型是否已构建？[[业务分析框架——附件1]] |      |
| 8   | 飞轮定位  | 这个产品在农场飞轮中贡献什么？（引流/信任/复购/利润/传播，五选一主定位）                                     |      |
| 9   | 系统贡献  | 即使自身不盈利，它为农场其他产品贡献了什么？这个贡献是否可被替代？                                          |      |
| 10  | 底线承受  | 最坏情况是什么？发生了能承受吗？如果不能，预防措施是什么？                                              |      |

**设计逻辑**：这10项是产品的“出生证”。用户不存在、需求不真实、没差异化、没能力交付、在飞轮中没位置、最坏情况承受不了——只要有一项站不住，这个产品就不该做。表A的价值就是在这里把烂想法拦住，不给它进入详细规划的机会。

### 表B：产品落地规划表（深度执行——怎么做）

> **用途**：表A全部通过后，用本表做详细落地设计。每个产品独立填写一份。  
> **与表A的关系**：表A的10项结论（用户、需求、差异化、飞轮定位、系统贡献、底线承受等）直接作为本表的输入，不重复填写。本表聚焦“怎么把这个产品做出来、卖出去、管起来”。

| 分析项          | 内容                                            | 必填/可选 |
| ------------ | --------------------------------------------- | ----- |
| **基础信息**     |                                               |       |
| 产品名称         |                                               | 必填    |
| 所属阿米巴        |                                               | 必填    |
| 产品层级         | 引流款 / 核心款 / 利润款 / 锚点款（四选一）                    | 必填    |
| 产品描述         | 用一段话说清楚：这个产品是什么？客户得到什么？体验流程是怎样的？              | 必填    |
| **客群与需求**    |                                               |       |
| 目标客群画像       | 基于表A的用户定义展开：年龄、家庭结构、消费能力、使用场景                 | 必填    |
| 客户决策逻辑       | 客户为什么买？第一推动力是什么？他们现在的替代方案是什么？                 | 必填    |
| **定价与财务**    |                                               |       |
| 定价           | 单价及定价逻辑（成本加成/市场对标/价值定价）                       | 必填    |
| 预期销量         | 月均销量，必须区分旺季和淡季                                | 必填    |
| 月均收入         | 单价 × 月均销量（分旺季/淡季）                             | 必填    |
| 主要成本项        | 原材料、人工、包装、运输、营销等                              | 必填    |
| 毛利率          | （收入 - 变动成本）/ 收入                               | 必填    |
| 季节性应对        | 全年可售还是仅限某几个月？淡季是否有替代产品或变体？                    | 必填    |
| **复购与传播**    |                                               |       |
| 复购机制         | 客户为什么第二次、第三次购买？内置的复购钩子是什么？（周期性消耗？养成回访？会员权益？）  | 必填    |
| 传播机制         | 客户会不会主动传播？触发点是什么？（拍照点？故事感？社交货币？）              | 必填    |
| **产品关联**     |                                               |       |
| 与阿米巴内其他产品的关系 | 是独立销售，还是和其他产品有捆绑/连带/升级关系？                     | 必填    |
| 与农场其他阿米巴的关系  | 承接哪个上游阿米巴/产品的客户和流量？输送给哪个下游阿米巴/产品？（基于表A飞轮定位展开） | 必填    |
| **交付与标准化**   |                                               |       |
| 交付流程         | 从下单到服务完成的完整步骤，5-8步描述清楚                        | 必填    |
| 关键触点         | 流程中决定客户体验成败的触点（不超过3个）                         | 必填    |
| 人员需求         | 交付需要几个人？什么技能？全职还是兼职？                          | 必填    |
| 标准化程度        | 交付多大程度可SOP化？多大程度依赖人的经验和热情？（百分比估算）             | 必填    |
| 关键岗位依赖度      | 关键执行人离职，产品能否在1周内由他人接手？需要什么条件？                 | 必填    |
| **数据与决策**    |                                               |       |
| 核心数据项        | 为衡量产品成功，需要采集哪些数据？                             | 必填    |
| 核心KPI        | 这个产品最重要的3个指标是什么？                              | 必填    |
| 退出标准         | 什么量化条件下，这个产品应该下架或暂停？                          | 必填    |

**设计逻辑**：

- 表A已经确认了“用户是谁、需求是什么、差异化在哪、飞轮定位是什么、底线能不能承受”，表B不再重复，直接基于这些结论展开执行细节
    
- 表B聚焦五件事：卖给谁（客群画像）→ 怎么定价（财务）→ 怎么让人反复买（复购）→ 怎么让人主动传（传播）→ 怎么交付（标准化）
    
- 退出标准是强制项——上线前就想好怎么下架

## 框架使用说明

### 使用方式

1.  **先做阿米巴总体分析**（第一至第七部分），明确该阿米巴在飞轮中的定位
2.  **列出产品清单**，确定该阿米巴下有哪些产品，各自属于哪个层级
3.  **对每个产品做独立分析**（使用产品分析子框架）
4.  **汇总所有产品的财务数据**，回填到阿米巴总体分析的第四部分，完成阿米巴损益表
5.  **所有阿米巴分析完成后，做一次飞轮整合**——检查各阿米巴之间的“承接谁”和“输送谁”是否一一对应，是否形成闭环

### 与其他文档的关系

-   本框架产出的各阿米巴分析方案，与《青松农场经营宏观分析》《劳动换薪策略档案》《客户视角分析》共同构成农场经营体系的核心文档
-   各阿米巴方案是“术”层面的落地文件，宏观分析是“道”层面的指导原则
-   AI经营系统（Hermes/hindsight/gbrain）是执行和迭代这些方案的工具载体

### 框架的迭代

-   每个阿米巴运营满6个月后，用实际数据回填框架，与当初的预测对比
-   对比差异超过30%的指标，分析原因，更新假设
-   框架本身每年迭代一次，删掉没用的分析项，增加实战中发现的新维度


## 框架背后的核心逻辑

1.  **第一性原理**：农场经营的本质是修复人与自然的关系。每个阿米巴、每个产品的分析，都要回到“这让人离自然更近还是更远”这个标尺上
2.  **飞轮思维**：阿米巴不是孤岛，是飞轮上的齿轮。分析一个阿米巴，必须看它和上下游的承接与输送
3.  **阿米巴核算**：独立核算但不独立存在。内部转移定价是协作的制度保障
4.  **系统驱动**：标准化程度是衡量阿米巴健康度的关键指标。对人的依赖越低，阿米巴越健康
5.  **最坏情况推演**：不赌概率，赌承受能力
6.  **量化退出标准**：在进入之前就想好怎么退出
7.  **双层分析**：阿米巴层级看系统角色，产品层级看市场竞争力。两层互相印证，才能既不失位、又不失血
