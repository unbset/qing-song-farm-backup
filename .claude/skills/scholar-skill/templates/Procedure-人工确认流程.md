# 📋 人工确认流程

**目的**: 确保 ScholarSkill 在关键决策点不偏离研究方向，保持与主人的长期目标一致。

---

## ⚠️ 触发确认的场景

### 必须确认（高优先级）

| 场景 | 说明 | 时限 |
|------|------|------|
| 创建新 MOC | 新增研究方向 | 24 小时内 |
| 标记核心论文 | 影响知识权重 | 48 小时内 |
| 方向调整建议 | 改变研究焦点 | 72 小时内 |
| 认知冲突发现 | 可能修正现有认知 | 48 小时内 |
| 删除重要笔记 | 不可逆操作 | 执行前 |

### 建议确认（中优先级）

| 场景 | 说明 | 时限 |
|------|------|------|
| 提取 >5 个新概念 | 知识爆发期 | 周度反思 |
| 月度方向反思 | 定期回顾 | 月末 |
| 风险识别 | 潜在问题 | 周度反思 |

### 无需确认（低优先级）

- 常规论文笔记创建
- 单个概念/洞察卡片创建
- MOC 内容更新
- 常规周度反思

---

## 🔄 确认流程

```text
1. ScholarSkill 识别需要确认的事项
   ↓
2. 创建确认请求笔记
   - 使用 Template-Confirmation-Request
   - 保存到 0-Inbox/
   - 标签：#confirmation-request #pending
   ↓
3. 通知主人
   - 飞书推送
   - 或等待主人查看
   ↓
4. 主人审查并决定
   - 查看请求笔记
   - 选择确认选项
   - 填写反馈
   ↓
5. ScholarSkill 执行决定
   - ✅ 同意 → 执行建议
   - 🔄 修改 → 按修改意见执行
   - ❌ 不同意 → 撤销/保持现状
   ↓
6. 更新状态并归档
   - 移动到 4-Outputs/Confirmation-Records/
   - 更新状态为 completed
```

## 🎯 确认决策原则

### ScholarSkill 提供建议时应遵循

1. **保守原则**: 不确定时倾向于维持现状
2. **证据原则**: 建议必须有论文或知识记录支撑
3. **可逆原则**: 优先选择可逆决策
4. **透明原则**: 清晰说明推理过程

### 主人决策时应考虑

1. **战略一致性**: 是否符合长期研究目标
2. **资源约束**: 时间与精力是否允许
3. **机会成本**: 选择这个意味着放弃什么
4. **灵活性**: 是否保留后续调整空间

## 🔧 自动化支持

### ScholarSkill 自动执行

1. 识别确认触发条件
2. 创建确认请求笔记
3. 发送飞书通知
4. 等待主人确认
5. 根据决定执行
6. 归档确认记录

### 需要人工介入

1. 审查确认请求
2. 做出决策
3. 提供反馈意见
